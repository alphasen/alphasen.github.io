# webpack笔记

  1. webpack是什么？

    webpack兼容（内部转换为webpack__require机制，其遵循了 commonJS规范）了js历史上所有的模块机制( commonJS，amd，cmd，es6)，将js代码处理为js引擎可以解析的代码

  2. webpack的作用？

    打包 处理依赖 优化

  3. webpack的优势？

    能被模块化的不仅仅是js了，其他各种类型的文件也可以通过loader处理成模块化

    扩展性强，插件机制完善

    我们可以直接使用require(xxx) 或者import …的形式来引入各种模块，而无需花费太多心思，因为webpack有着各种健全的加载器(loader) 在默默处理这些事情

  4. webpack结构构成

    // file webpack.config.js
const path=require('path')
const HtmlWebpackPlugin=require('html-webpack-plugin')
const CleanWebpackPlugin=require('clean-webpack-plugin')

module.exports={
  // entry:'./src/index.js',// 字符串和数组形式将会打包进一个output文件中
  entry:{
    index:'./src/index.js',
    index2:'./src/index2.js'
  },
  output:{
    path:path.resolve(__dirname,'/dist'),
    filename:'[name].js'
  },
  optimizion:{ // 优化选项
    minisize:false, // Tell webpack to minimize the bundle using the UglifyjsWebpackPlugin.
    splitChunks:{ // SplitChunksPlugin 配置项

    }
  },
  module:{ // 模块规则 各种loaders

  },
  pulugins:[ // 插件
    new HtmlWebpackPlugin({
      template:'./src/index.html', // 生成的html文件内容不是期望的内容的时候，检查或加上该配置
      title:'webpack-stydy',
      hash:true, // 在js的引入后面加入hash
      favicon:'path/to/favicon', // 标签页左侧的小图标
      minify:{
        collapseWhitespace:true, // 去除空格
        removeAttributeQuotes:true, // 去除属性的引号
      },
      chunks:['index','index2'], // 要引入的chunks，也就是entry中的key
    }),
    new CleanWebpackPlugin(), // clean output.path directory
    new CopyWebpackPlugin([{
            from:'./src/static',
            to:'./static', // 相对于 output的路径
    }]),
  ],
  devServer:{

  }
}
